# Starting with the Basics: Creating ADO Work Items

## Lesson 12

### Azure Board
- Helps teams plan, track and manage their work in an organized way.
- Its default columns have the same name *Work Items States*.
- You can full customize for any workflow.

#### Ways of use
+ **Kanban:** 
    + Is work managed as continuous flow.
    + There are no defined deadlines or formalities.
    + Best suited for Support Teams and Operations.
+ **Scrum:**
    + Structured framewor designed to help teams work more effectively.
    + Divide the workflow in set time periods called *sprints* (1 a 4 weeks, typically).
    + Ideal fo:
        + Product development or projects with evolving requirements (especially with frequent feedback demands). 
    + Values:
        + collaboration, transparency and continuous improvement.

#### Column Numbers
+ Called **WIP** (Work in progress).
+ Control how many work items can reside in a certain stage at one time.
+ When the limit is exceeded, the column simply is highlighted to red.


## Lesson 13

### Epic
Represents the big picture, the overarching initiative that guides a program or project.

#### How to create?
+ Boards/ Work items/ New Work Item
+ Boards/ Backlogs/ New Work Item
+ Boards/ Boards/ New Column/ + New item

#### Item Properties
+ Id
+ Name 
+ User assigned (Generally to a product owner or manager)
+ Coments
+ Tags (for segregating work)
+ State --> Reason
+ Area: Represents the functional area or team responsible for the epic.
+ Iteration: Defines the epic timeline or sprint.

#### Item Details Fields
+ Description
+ Discussion
+ Planning
    + Priority
    + Risk
    + Effort 
    + Business Value
    + Time Criticaly
    + Dates (Start, Target)
+ Classification
+ Deployment
+ Development
+ Related Work 

#### Item Tabs
Details, History, Link, Files, Configurations

#### Item Follow Button
Apply notifications

## Lesson 15

### Anatomy of a User Story (US)
+ A US describes a feature or funtionality from an end-user´s perspective.
+ In contrast to a US, a requirement is a statement that specifies what a
    system or product must do or how it must behave to meet specific
    criteria.
+ Key statement:
    + **'As a'** represents the Persona.
    + **'I want'** specifies the desired capability.
    + **'So that'** defines the value or purpose behind the requested capability
                    for the user.

### Acceptance Criteria
Confirms that we built the right thing for the right user, with the right value in mind.
