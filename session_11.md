# Managing Projects with Azure DevOps (ADO)

## Lesson 115

### Stages of Team Formation (Tuckman Model)

1. Forming
2. Storming (conflicts emerge)
3. Norming (conflicts are resolved)
4. Performing
5. Adjourning (project ending)

## Lesson 116

### Scrum Process Flow

**Product Backlog** → **Sprint Planning** → **Sprint Backlog** → **Sprint Execution** → **Sprint Review** → **Potentially Shippable Increment**
                                                                ^
                                                                |
                                                        **Sprint Retrospective**



### Scrum Ceremonies

1. Sprint
2. Sprint Planning
3. Daily Scrum (Daily Stand-up / Scrum of Scrums)
4. Sprint Review
5. Sprint Retrospective

## Lesson 117

### Definition of Done (DoD)

Defines what constitutes the completion of a work item. It outlines the shared, team-wide criteria for when the work is fully finished and meets the expected quality standards.
